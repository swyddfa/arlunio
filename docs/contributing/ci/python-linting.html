Python Linting¶

This action runs all our static analysis tools against the codebase as defined in our .pre-commit-config.yaml file.

Triggers¶

This action is triggered by any PR that is opened against the develop or master branches, unless the changes only affect the blog.

on:
  pull_request:
    branches:
    - develop
    - master
    paths-ignore:
    - 'blog/**'

Jobs¶

Since static analysis tools only look at the source code without importing or running anything, it doesn’t matter too much which platform / python version they are run on as long as the tool itself supports it. For the sake of simplcity these checks are run on the latest version of python and on a linux agent.

  Lint:
    runs-on: ubuntu-latest

Steps¶

    - uses: actions/checkout@v1

    - name: Setup Python 3.7
      uses: actions/setup-python@v1
      with:
        python-version: 3.7

    - name: Setup Environment
      run: |
        python --version
        python -m pip install --upgrade pip
        python -m pip install --upgrade tox

    - name: Tox
      shell: bash
      run: |
        tox -e lint

This action is fairly straight forword, we checkout the code, setup the python envrionment and then run our lint tox envrionment which handles the details of running the static analysis tools via pre-commit
